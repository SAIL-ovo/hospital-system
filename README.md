# 社区医院管理系统
个人毕设项目
一个面向社区医院场景的前后端分离管理系统，覆盖居民预约挂号、医生接诊、在线问诊、电子病历、药品库存、健康自评与后台管理等核心业务。

项目适合用于：

- 社区医院信息化课程设计
- Spring Boot + Vue 综合实训项目
- 基层医疗服务系统原型与二次开发

## 项目简介

本系统围绕社区医院日常业务进行设计，提供三类主要使用角色：

- 普通用户：预约挂号、在线问诊、查看健康档案、健康自评
- 医务人员：查看待诊预约、处理问诊、记录病历、查看药品库存
- 管理员：管理医生、用户、药品、轮播图、预约及系统基础数据

系统目标是将社区门诊、居民健康服务与基层医疗管理整合到统一平台中，提升居民就诊体验与医院业务协同效率。

## 数据库设计概览

根据当前项目表结构，核心表包括：

- `admin`：管理员信息
- `user`：普通用户
- `doctor`：医生信息
- `appointment`：预约挂号
- `consultation`：在线问诊
- `medical_record`：电子病历
- `goods`：药品信息
- `category`：药品分类
- `carousel`：首页轮播图
- `health_assessment_question`：健康自评题库
- `user_health_assessment_record`：用户健康自评记录

主要业务关系：

- 用户与医生通过 `appointment` 建立预约关系
- 用户与医生通过 `consultation` 建立问诊关系
- 预约可生成电子病历 `medical_record`
- 药品 `goods` 隶属于药品分类 `category`

## 后端启动

### 1. 配置数据库

修改后端配置文件：

- [application.yml](E:/the%20%20hospital%20of%20%20community/code2026/springboot/src/main/resources/application.yml)

重点确认：

- 数据库地址
- 数据库用户名密码
- 端口配置

### 2. 初始化数据库

将你的建表 SQL 导入 MySQL，并准备基础测试数据。

### 3. 启动 Spring Boot

在后端目录执行：

```bash
cd springboot
mvn spring-boot:run
```

或在 IDE 中直接运行：

- `SpringbootApplication.java`

## 前端启动

在前端目录执行：

```bash
cd vue
npm install
npm run dev
```

生产打包：

```bash
npm run build
```

## 当前已知注意事项

基于当前仓库开发过程中已发现的情况，建议注意：

- 前端部分页面存在中文编码历史问题，修改文件时建议统一为 UTF-8
- 个别页面/路由若引用缺失组件，需要补齐后再做完整构建
- 统计类卡片应优先按业务 `key` 更新，避免用数组下标造成显示错位
